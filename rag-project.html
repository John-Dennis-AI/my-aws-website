Private RAG with Cost Controls

John Dennis · RAG · AWS · Serverless
Overview

This project demonstrates a Retrieval-Augmented Generation (RAG) pipeline on AWS designed for private data, predictable costs, and fast answers. It indexes documents in Amazon S3, retrieves relevant chunks, and generates grounded responses via an LLM (e.g., Amazon Bedrock), with guardrails for latency and spend.

TL;DR: S3 → (ingest/chunk/embed) → vector index → retrieve + rerank → Bedrock LLM → response

Tech Stack & Decisions
  • Chunking: e.g., 800–1,000 tokens, 10–15% overlap (keeps context intact).
  • Embeddings: Bedrock or open-source model (consistent across index/query).
  • Vector Index: OpenSearch vector engine or FAISS (low-cost demos).
  • Retrieval: top-k = 4–8; optional MMR to diversify similar chunks.
  • Reranking: optional lightweight reranker for quality ↑ with small latency cost.
  • LLM: Amazon Bedrock model family; prompts enforce citations & refusal rules.
  • Observability: CloudWatch metrics for latency, token counts, and hit rate.
Cost & Performance (Demo Targets)
  • Indexing cost: one-time per doc batch; keep small for a portfolio demo.
  • Query latency: target < 1.5s p50 (cached embeddings + simple prompts).
  • Monthly run cost: $$ only when you query; this site is static (no public API).

Note: Public site is static—no endpoints exposed. Live queries remain private in my dev account.

Security & Privacy
  • No PII in the demo corpus; strict IAM roles; secrets stored in Parameter Store.
  • No public write endpoints; all data at rest encrypted; logs scrubbed of sensitive tokens.
