Agentic Workflow on AWS

John Dennis · Agentic Systems · AWS · Bedrock
Overview

This project showcases an agentic workflow built with AWS services. An LLM (Amazon Bedrock) acts as the reasoning engine, orchestrated by AWS Step Functions, and supported by Lambda functions that perform specific tasks (APIs, data lookups, business logic).

TL;DR: Bedrock Agent → Step Functions → Lambda Tasks → Results

Workflow & Tools
  • Bedrock Agent: reasoning core, decides which step/tool to invoke.
  • Step Functions: orchestrates calls to Lambda functions safely.
  • Lambda Functions: tools for API calls, database queries, or data transformations.
  • S3: used to store intermediate artifacts and logs.
  • IAM: strict roles/permissions to enforce least privilege.
Use Case Example

The agent is tasked with customer order tracking:

  1. User asks: “Where’s my package?”
  2. Bedrock Agent parses intent → decides to call Order Lookup Lambda.
  3. Step Functions run the Lambda with the order ID.
  4. Result returned → Agent formulates natural-language response with status + ETA.
Cost & Security
  • Pay-per-use only (Lambda & Step Functions scale to zero when idle).
  • No public endpoints exposed; API Gateway is private to the demo account.
  • All secrets managed in AWS Parameter Store.
